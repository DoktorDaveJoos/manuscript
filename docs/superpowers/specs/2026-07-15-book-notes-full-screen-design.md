# Book Notes Full-Screen Design

## Goal

Make the book-wide Notes & Research page use the same full-height workspace pattern as the chapter editor while preserving the book sidebar and all existing note editing behavior.

## Layout

- Keep the root `h-screen` application shell and the existing book `Sidebar`.
- Replace the padded, vertically scrolling, `max-w-[760px]` content column with a `flex min-w-0 flex-1 flex-col overflow-hidden` main workspace.
- Render `NotesPanel` directly inside that workspace so it fills all remaining width and height.
- Remove the page-level `PageHeader` and `Card`. `NotesPanel`'s existing `PanelHeader` becomes the single compact page header and continues to expose the notebook title, save state, and keyboard hints.

## Shared UI Requirements

- Continue using `@/components/ui/Button` for every notes action, including slash-menu choices and table actions.
- Do not introduce raw button markup or a second button implementation.
- For any future notes control, reuse an existing component from `resources/js/components/ui/`; if none fits, evaluate the matching shadcn component before writing custom markup.
- Preserve project design tokens and existing shared inputs, alerts, checkbox, keyboard hint, spinner, and panel header components.

## Behavior and Data Flow

The change is presentational only. Preserve:

- the book-wide notes save endpoint generated by Wayfinder;
- initial note content and optimistic version handling;
- autosave and visible save/error states;
- slash-command behavior, markdown blocks, todos, dividers, callouts, and tables;
- the book-wide `5_000_000` character limit;
- existing navigation through the persistent book sidebar.

## Error Handling

No error contract changes. Save failures and character-limit failures continue to render through the existing shared `Alert` surface inside `NotesPanel`.

## Verification

- Run the focused `tests/Browser/BookNotesTest.php` browser test.
- Open the Notes & Research route in a real browser and verify that the notes surface fills the viewport beside the sidebar without page-level vertical scrolling.
- Confirm the browser reports no JavaScript errors.
- Confirm notes controls are rendered through shared UI components and no raw `<button>` is introduced in the notes page, panel, or slash menu.
