# Amazon KDP (Kindle Direct Publishing) — Comprehensive Analysis

*Research compiled March 2026*

---

## 1. What KDP Offers for Free

Amazon KDP is a free-to-use self-publishing platform. Authors pay nothing upfront — Amazon takes its cut from royalties on each sale. The platform provides:

### Publishing Capabilities
- **eBooks** — Kindle format, distributed to all Kindle devices and apps worldwide
- **Paperbacks** — Print-on-demand, no inventory required
- **Hardcovers** — Print-on-demand (added more recently)
- **Audiobooks** — Via sister platform ACX (separate signup)

### Free Tools

| Tool | What It Does | Limitations |
|------|-------------|-------------|
| **Kindle Create** | Desktop app (Mac/Windows) for formatting manuscripts into KPF files. Import a .doc/.docx, apply themes, generate TOC, preview across devices. | Not a writing tool — only formats completed manuscripts. Only exports KPF (Amazon-only format). Cannot go "wide" to other retailers. Limited to 4 themes. No custom fonts, margins, or page separators. No image captions. No tables/footnotes in paperback output. Produces bloated file sizes. |
| **Cover Creator** | Browser-based tool for designing basic book covers using templates and stock images. | Generic templates that thousands of authors use. Very limited customization. Designs using stock images cannot be downloaded or used outside KDP (no marketing materials, social media, etc.). Does not support Japanese, Hebrew, or Yiddish. |
| **Kindle Previewer** | Desktop app to preview how eBooks render on tablets, phones, and e-ink readers. | Preview only — no editing capability. |
| **Online Previewer** | Browser-based preview tool on the KDP Bookshelf. | Less thorough than the desktop previewer. |
| **Print Previewer** | Checks paperback/hardcover files for errors after upload. | Catches errors but doesn't fix them. |
| **Cover Calculator & Templates** | Generates measurements and provides downloadable templates for print covers (spine width, bleed, margins). | Just measurements — no design capability. |
| **File Setup Calculator** | Excel tool providing margin, bleed, and spine measurements for paperback interiors. | — |
| **Printing Cost & Royalty Calculator** | Estimates printing costs and royalties before publication. | — |

### Promotional Features (with KDP Select enrollment)
- **Kindle Unlimited** — Readers with KU subscription can read for free; authors earn per page read
- **Kindle Countdown Deals** — Time-limited promotional pricing while still earning royalties
- **Free Book Promotion** — Make your book free for a limited period
- **Up to 70% royalty** on eBooks priced $2.99–$9.99 (35% outside that range)

### Recent Changes (2025–2026)
- **DRM-free EPUB/PDF downloads** (Jan 2026) — Authors who set titles as DRM-free now have books available in EPUB and PDF formats, readable on Kobo, NOOK, Apple Books, etc.
- **MOBI deprecated** (March 2025) — KDP no longer accepts MOBI files for fixed-layout eBooks
- **Print royalty cut** (June 2025) — Paperback/hardcover royalties dropped from 60% to 50% for books priced below certain thresholds

## 2. KDP's EPUB/Manuscript Creation Capabilities

### Critical Point: KDP Has NO Writing Editor

**KDP does not provide any authoring, writing, or editing tools.** There is no built-in text editor, no manuscript creation capability, and no way to write a book within the KDP ecosystem. Authors must:

1. Write their manuscript in a separate application (Word, Google Docs, Scrivener, etc.)
2. Export as .docx (or other supported format)
3. Either upload directly to KDP, or import into Kindle Create for formatting
4. Upload the formatted file to KDP for publishing

### Kindle Create — Formatting Only, Not Authoring

Kindle Create is explicitly a **formatting tool**, not a writing tool. Its workflow:
- Import a completed .doc/.docx manuscript
- Apply one of ~4 visual themes
- Auto-generate a Table of Contents
- Add chapter headings, section breaks, drop caps
- Preview across device types
- Export a .KPF file for upload to KDP

**What Kindle Create cannot do:**
- Write or compose text (no text editor)
- Edit prose content (only formatting adjustments)
- Create manuscripts from scratch
- Export to EPUB, PDF, or any non-Amazon format
- Create box sets / omnibus editions
- Customize fonts beyond the few provided
- Adjust print margins
- Handle tables, footnotes, or endnotes in paperbacks
- Support image captions
- Handle full-bleed images in paperbacks
- Produce compact file sizes (known for bloated output)

### Accepted Upload Formats

**For eBooks:**
- **EPUB** (recommended) — cleanest conversion
- **KPF** (Kindle Create output) — guaranteed compatibility
- **DOC/DOCX** — acceptable but conversion can be unpredictable
- HTML, RTF, TXT — supported but not recommended
- PDF — limited language support (English, French, German, Italian, Portuguese, Spanish, Catalan, Galician, Basque only)
- ~~MOBI~~ — deprecated as of March 2025

**For Print Books:**
- DOC/DOCX, PDF, HTML, RTF

## 3. KDP's Limitations — What It Does NOT Do Well

### No Authoring or Editing Tools
- No writing environment whatsoever
- No grammar checking, spell checking, or proofreading
- No AI-assisted editing or suggestions
- No collaborative editing
- No version history or revision tracking
- Authors are entirely on their own for content creation

### Formatting Pain Points
- **DOCX conversion is unreliable** — Manual formatting (line breaks, extra returns, tabs) causes text flow problems. Pasted web content carries invisible markup that breaks conversion.
- **Images frequently break** — Graphics used for separators get distorted (aspect ratio removed). Images may display wrong size or disappear entirely. Must use RGB JPG/PNG at specific resolutions (300–600px inline, 1200–2500px full-width).
- **Custom fonts often fail** — May not render, may reflow oddly, or get rejected for licensing reasons.
- **Structural EPUB errors** — Incorrect manifest entries, broken spine order, unsupported MIME types cause rejections. Authors must run EPUBCheck externally.
- **Chapter navigation breaks** — If headings aren't properly tagged (H1/H2), the TOC doesn't work.
- **Metadata problems** — Can be overwritten or lost during conversion.

### Print Quality Issues (Widely Reported)
- Pages falling out of bindings ("four random pages fell out in the first few chapters")
- Wrong book contents shipped with correct covers
- Interior illustrations distorted with "white vertical lines"
- Inconsistent quality between proof copies and retail versions
- Smudged covers and damaged hardback covers
- Author copies taking 2–6 weeks to arrive

### Account & Business Issues
- **Sudden account terminations** without clear explanation or meaningful appeal process
- **Royalty withholding** — Funds frozen when accounts are suspended
- **Customer service is almost non-existent** — Automated bot responses, no human review, generic repetitive replies
- **False copyright claims** — Legitimate authors blocked for "similarity" to other contributor names
- **Vague policy enforcement** — "Activities trying to manipulate our services" with no specifics
- **No transparency** — KDP refuses to reveal detection technology or specific grounds for refusal

### Platform & Ecosystem Limitations
- **Amazon-only ecosystem** — KDP Select requires eBook exclusivity; Kindle Create only exports to Amazon format
- **Declining organic visibility** — Algorithm now prioritizes books with ad spend and strong early conversion
- **Rising ad costs** — Pay-to-play dynamic, especially in crowded niches
- **Daily upload limit** — Maximum 3 new titles per day (anti-AI-spam measure)
- **Content review delays** — 3–10 business days for new titles to go live
- **No direct-to-consumer sales** — Authors cannot sell directly; Amazon controls the customer relationship
- **No built-in marketing tools** — Beyond basic KDP Select promotions
- **No wide distribution** — Cannot reach indie bookstores, libraries, or other retailers (unlike IngramSpark)
- **264% increase in new titles** over 5 years — massive competition for attention

### What Authors Must Source Elsewhere
KDP does not offer and authors must independently find:
- Writing software
- Professional editing (developmental, copy, line editing)
- Professional cover design
- Interior formatting beyond Kindle Create's basics (Vellum $249, Atticus $147)
- Marketing and advertising strategy
- ISBN management (KDP provides free ASINs but not ISBNs)
- Multi-platform distribution
- Email list building and reader engagement tools

## 4. KDP's Formatting Requirements — Common Pain Points

### Manuscript Formatting Rules
- **Recommended image specs:** Cover minimum 1000px on longest side (2560x1600 ideal); interior images 300 DPI, RGB color mode
- **File size limit:** 650 MB for eBooks
- **Supported languages for PDF upload:** Only 9 languages
- **No fixed-layout MOBI** as of March 2025
- **Maximum 10 titles per format weekly** (in addition to 3/day limit)

### Common Formatting Disasters

| Problem | Cause | Pain Level |
|---------|-------|------------|
| Paragraph indents all over the place | Mixed formatting in source DOCX | Very High |
| Images disappear or resize wrong | CMYK color profiles, wrong resolution, broken paths | High |
| Table of Contents doesn't work | Headings not tagged as H1/H2 | High |
| Custom fonts replaced with defaults | Font licensing issues or embedding failures | Medium |
| Random blank pages appear | Section break handling differences | Medium |
| Line separators/ornaments distorted | KDP removes fixed aspect ratio on graphics | Medium |
| Footnotes/endnotes break | Conversion doesn't handle note linking reliably | High |
| Text reflows unpredictably | Invisible web markup, manual formatting | High |

### The "Formatting Tax"
Many authors report spending more time fighting formatting than writing. The typical workflow requires:
1. Write in Word/Scrivener/Docs
2. Export to DOCX
3. Import to Kindle Create (Amazon only) OR format in Vellum/Atticus ($150–$250)
4. Export to KPF or EPUB
5. Upload to KDP
6. Preview and find formatting errors
7. Go back to step 1–3 to fix issues
8. Re-upload and re-preview
9. Repeat until acceptable

This cycle is where tools like Vellum ($249, Mac only) and Atticus ($147, cross-platform) have built businesses — by making the formatting step painless and producing clean output for both Amazon and wide distribution simultaneously.

## 5. AI Features KDP Offers

### KDP Offers ZERO AI Features

**Amazon KDP does not provide any AI-powered tools for authors.** There is:
- No AI writing assistance
- No AI editing or proofreading
- No AI-powered formatting suggestions
- No AI cover generation
- No AI metadata/description optimization
- No AI-powered marketing tools
- No AI content analysis or feedback

### KDP's AI Policy (What Authors Must Disclose)

KDP has established disclosure requirements around AI but offers nothing itself:

**Must Disclose ("AI-Generated"):**
- Text drafted by AI models (even with substantial edits afterward)
- Cover art or interior illustrations rendered by AI image generators
- Machine translations used as the final published edition
- Activity books, descriptions, or summaries created by AI tools

**No Disclosure Required ("AI-Assisted"):**
- Grammar and spelling checks
- Clarity rewrites where author maintains control
- Brainstorming plot points or outlines
- Style suggestions
- Using AI to spark ideas (which the author then writes)

**The Key Test:** "Who produced the first version of the words or pixels that made it into the book?" If AI produced the first version, disclose.

### AI Content Enforcement
- Authors must select "AI-generated" during title setup if applicable
- Failure to disclose may result in book removal or account warnings
- Quality requirements: AI-generated books must involve "significant human editing"
- Authors remain responsible for fact-checking, originality, and structure
- Books flagged as low-quality AI content may be suppressed or removed

### Anti-AI-Spam Measures
- 3 titles/day upload limit (introduced to combat AI-generated book flooding)
- Frequent releases may trigger originality verification
- Enhanced content review for suspected AI-generated content

## Summary: The KDP Gap Analysis

| Capability | KDP Provides | Gap for Authors |
|-----------|-------------|-----------------|
| **Writing/Authoring** | Nothing | Must use separate software entirely |
| **Editing** | Nothing | Must hire editors or use separate tools |
| **AI Assistance** | Nothing | Must use third-party AI tools, then disclose |
| **Formatting** | Basic (Kindle Create) | Limited, Amazon-only output, many pain points |
| **Cover Design** | Basic (Cover Creator) | Generic templates, can't use designs outside KDP |
| **Preview** | Good (Kindle Previewer) | Preview only, no fix capability |
| **Distribution** | Amazon only | No wide distribution, no bookstores/libraries |
| **Print Quality** | Inconsistent | Well-documented binding/printing defects |
| **Customer Service** | Minimal/automated | Bot responses, no meaningful human support |
| **Marketing** | Basic promotions | No real marketing or discovery tools |
| **Analytics** | Sales dashboard | Basic compared to third-party tools |
