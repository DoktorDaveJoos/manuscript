# Editor Default Settings & Formatting Controls Research

**Date:** 2026-03-24
**Purpose:** Document the exact default editor settings, typography choices, and formatting controls offered by the leading writing/manuscript apps. Synthesize into actionable recommendations for Manuscript's editor.

---

## App-by-App Default Settings

### 1. Scrivener 3 (Literature & Latte)

| Setting | Default Value |
|---|---|
| **Font** | Palatino Regular |
| **Font size** | 13pt |
| **Line spacing** | 1.2-1.3x (multiple) |
| **Paragraph spacing** | 0pt before, 0pt after (configurable) |
| **First-line indent** | ~0.5" (18pt) |
| **Paragraph model** | First-line indent (no extra spacing) |

**Formatting toolbar controls:**
- Font family picker
- Font weight picker (Regular, Italic, Bold, Bold Italic)
- Font size selector
- Bold (B), Italic (I), Underline (U) buttons
- Text alignment (left, center, right, justify)
- Line spacing dropdown
- Indent controls
- Styles dropdown (No Style, Block Quote, Heading, etc.)
- Highlight and text color
- Lists (bulleted, numbered)

**Focus/distraction-free features:**
- **Composition Mode** (Cmd+Opt+F): Full-screen distraction-free writing with customizable background color/texture. Only a minimal control bar appears when hovering at screen bottom.
- **Typewriter Scrolling** (Cmd+Ctrl+T): Keeps the active line vertically centered. On by default in Composition Mode, off by default in the regular editor.
- **Focus Mode** (View > Text Editing > Focus): Three granularity levels -- Line, Sentence, or Paragraph. Dims all text outside the selected scope to gray.

### 2. Ulysses

| Setting | Default Value |
|---|---|
| **Font** | System font (San Francisco on Apple platforms) |
| **Font size** | System default (~16px equivalent), adjustable via Text Zoom slider |
| **Line height** | ~1.5x (adjustable slider) |
| **Paragraph spacing** | Visible vertical whitespace (adjustable slider) |
| **First-line indent** | 0 (off by default; adjustable slider available) |
| **Paragraph model** | Block spacing (Markdown-native) |

**Formatting controls (Markup Bar):**
- Context-sensitive markup bar at bottom of editor
- Headings (# through ######: Title, Heading, Subheading)
- Bold, Italic, Strikethrough, Marked (highlight)
- Links and images
- Ordered and unordered lists
- Block quotes
- Code (inline and block)
- Divider / horizontal rule
- Footnotes, annotations, comments
- No text alignment controls (Markdown does not support alignment)
- No font/size controls in toolbar (set globally in preferences)

**Focus/distraction-free features:**
- **Typewriter Mode** with three sub-options:
  - **Highlight**: Dims surrounding text to focus on current sentence/paragraph
  - **Fixed Scrolling**: Keeps the typing line vertically fixed on screen
  - **Mark Current Line**: Subtle highlight on the active line
- **Editor Focus** (Mac): Fades the toolbar when in windowed mode
- **Hide Interface**: Removes even window controls for pure full-screen writing
- **Dark mode** with multiple editor themes

### 3. iA Writer

| Setting | Default Value |
|---|---|
| **Font** | iA Writer Mono (custom monospace, based on IBM Plex) |
| **Font size** | Dynamically calculated; ~18px equivalent on desktop |
| **Line height** | Auto-calculated based on font size and column width; typically ~1.6-1.7x for mono |
| **Paragraph spacing** | Visible vertical whitespace (Markdown: blank line = new paragraph) |
| **First-line indent** | 0 (never in editor; available as export/preview toggle) |
| **Paragraph model** | Block spacing only (indent is explicitly an output concern) |
| **Line width** | 64 characters per line before wrapping |

**Font options (3 custom typefaces):**
- **iA Writer Mono**: Monospace (default). Steady rhythm, reduces editing impulse.
- **iA Writer Duo**: Duospace. Proportional-feel with monospace regularity. More natural reading.
- **iA Writer Quattro**: Proportional with monospace influence. Closest to reading a book.

**Formatting controls:**
- Headings (H1-H6 via Markdown #)
- Bold, Italic
- Links, images
- Ordered and unordered lists
- Block quotes
- Code (inline and fenced blocks)
- Footnotes
- Tables (via Markdown)
- Task lists (checkboxes)
- No underline, no strikethrough in toolbar (Markdown limitation)
- No text alignment controls
- No font/size/color controls in toolbar

**Focus/distraction-free features:**
- **Focus Mode** (pioneer of this concept) with three sub-modes:
  - **Sentence**: Highlights active sentence, dims everything else
  - **Paragraph**: Highlights active paragraph, dims everything else
  - **Typewriter**: Cursor stays vertically centered; no dimming
- **Syntax Highlighting for prose**: Colors parts of speech (adjectives, nouns, adverbs, verbs, conjunctions) to catch overwriting
- **Full-screen writing** with minimal chrome
- **Dark mode**

- No font size control per block

- **Full-width mode**: Removes side margins for wider text area
- No focus mode, typewriter mode, or text dimming

## Industry-Standard Manuscript Defaults (Creative Writing)

Per submission guidelines from agents, publishers, and organizations like the Chicago Manual of Style:

| Setting | Standard Value |
|---|---|
| **Font** | Times New Roman (or Courier New) |
| **Font size** | 12pt |
| **Line spacing** | Double-spaced (2.0) |
| **Margins** | 1" on all sides |
| **First-line indent** | 0.5" |
| **Paragraph spacing** | 0pt (no extra space between paragraphs) |
| **Alignment** | Left-aligned (ragged right), never justified |
| **Scene breaks** | Centered "#" or "***" on its own line |

**Key conventions:**
- First paragraph after a chapter title or scene break: flush left (no indent)
- All subsequent paragraphs: 0.5" first-line indent
- Italics used for emphasis (no underlining in modern submissions)
- One space after periods (not two)

## Synthesis: Essential vs. Nice-to-Have vs. Omit

### ESSENTIAL (Must have for any manuscript/book writing app)

1. **Bold** -- Universally available. Required for emphasis in notes, synopses, and non-fiction.
2. **Italic** -- The single most important character style for fiction. Used for internal thoughts, emphasis, foreign words, titles.
3. **Headings** (at least H1-H3) -- Chapter titles, scene headers, structural organization.
4. **Paragraph handling** -- Clear visual separation between paragraphs (block spacing in editor).
5. **Word/character count** -- Every writing app offers this. Writers track progress obsessively.
6. **Undo/redo** -- Fundamental editing.

### HIGHLY RECOMMENDED (Expected by serious writers)

7. **Bulleted and numbered lists** -- Useful for outlines, notes, brainstorming within the editor.
8. **Block quote** -- Common in both fiction (epigraphs, letters) and non-fiction.
9. **Separator/scene break** -- Essential for fiction; denotes scene or POV changes.
10. **Link** -- References, research links within notes.
11. **Focus mode** (sentence or paragraph dimming) -- Offered by Scrivener, Ulysses, iA Writer, and Bear. Becoming a standard expectation for dedicated writing apps.
12. **Typewriter scrolling** -- Offered by Scrivener, Ulysses, iA Writer. Keeps writing position comfortable. Strongly associated with "serious writing tools."
13. **Dark mode** -- Universal expectation in 2026.

### NICE TO HAVE (Differentiators, not blockers)

14. **Strikethrough** -- Useful for editing/revision (crossing out text without deleting). Available in most apps.
15. **Underline** -- Rarely used in modern writing (italics replaced it). Omitted by Ulysses and iA Writer.
16. **Code blocks** -- Primarily for technical writing. Not needed for fiction.
17. **Task lists / checkboxes** -- More of a note-taking feature than a writing feature.
18. **Footnotes** -- Important for non-fiction; less so for fiction.
19. **Tables** -- Rarely needed in creative writing.
20. **Inline images** -- Nice for research integration; not needed in the manuscript itself.
21. **Highlight / mark** -- Useful for self-editing and revision passes.
22. **Syntax highlighting for prose** (parts of speech) -- Unique differentiator (iA Writer, Scrivener).

### SAFELY OMIT (Not needed or actively harmful for a focused writing app)

23. **Font picker in toolbar** -- Omitted by Ulysses, iA Writer, Bear, Notion. Font is a global preference, not a per-selection choice. Per-selection font changes lead to messy formatting.
24. **Font size in toolbar** -- Same reasoning. Size is a global editor setting.
25. **Text color** -- Not relevant for manuscript writing. Leads to formatting mess.
26. **Text alignment controls** -- Omitted by Ulysses, iA Writer, Bear, Notion. Manuscripts are always left-aligned. Alignment is an output/export concern.
27. **Superscript/subscript** -- Academic/scientific only.
28. **Borders and shading** -- Word processor feature, not a writing feature.
29. **Multiple font weights in toolbar** -- Beyond bold, extra font weight controls add complexity without value.
30. **Justify** -- Never used in manuscript submissions. Questionable even in final book formatting for ebooks.

## Recommended Defaults for Manuscript (Creative Writing Editor)

Based on the patterns across all apps surveyed:

### Editor Typography

| Setting | Recommended Default | Rationale |
|---|---|---|
| **Font** | System sans-serif (or a curated writing font like iA Writer Quattro style) | Matches Ulysses, Bear, Notion. System fonts render crisply on all displays. Alternatively, a curated serif like Charter or a purpose-built font signals "this is a writing tool." |
| **Font size** | 16-18px | The consensus range for comfortable screen reading. iA Writer uses ~18px, Bear/Notion use 16px. Scrivener's 13pt is universally considered too small. |
| **Line height** | 1.5-1.6x | The sweet spot for readability. Ulysses, Bear, and Notion all land here. Google Docs (1.15) and Word (1.08) feel cramped for extended writing. |
| **Paragraph spacing** | 0.75-1.0em after each paragraph | Visible but not excessive gap. Block paragraph model in editor. |
| **First-line indent** | 0 (in editor) | 80% of surveyed apps use no indent in the editor. Indent is an output/preview concern. |
| **Line width** | 60-75 characters | iA Writer uses 64. Optimal reading column is 45-75 characters per line (Bringhurst). Prevents eye fatigue. |
| **Alignment** | Left-aligned | Universal default. Never justify in the editor. |

### Minimal Formatting Toolbar

Based on the intersection of what Scrivener, Ulysses, iA Writer, and Bear all offer:

**Primary controls (always visible):**
1. Bold
2. Italic
3. Heading level (dropdown or cycle: H1/H2/H3)
4. Bulleted list
5. Numbered list
6. Block quote
7. Scene break / divider

**Secondary controls (overflow or keyboard-only):**
8. Strikethrough
9. Link
10. Highlight / mark
11. Footnote (if non-fiction is a use case)

**Keyboard shortcuts (no toolbar button needed):**
- Undo/redo (Cmd+Z / Cmd+Shift+Z)
- Bold (Cmd+B)
- Italic (Cmd+I)

### Focus Features (Priority Order)

1. **Typewriter scrolling** -- Highest impact, most distinctive of "real writing tools"
2. **Focus mode** (sentence-level dimming) -- Strong differentiator, loved by users of iA Writer and Scrivener
3. **Full-screen / distraction-free mode** -- Expected baseline feature
4. **Dark mode** -- Non-negotiable in 2026

## Key Takeaways

1. **The dedicated writing apps (Scrivener, Ulysses, iA Writer) converge on FEWER formatting controls than general-purpose tools (Word, Google Docs).** More formatting options is not better for a writing app -- it is actively worse. The writing apps succeed by removing choices, not adding them.

2. **Font and size are global settings, not per-selection controls.** Every Markdown-native writing app (Ulysses, iA Writer, Bear, Notion) enforces this. It prevents formatting soup and keeps the focus on content.

3. **Block paragraph spacing in the editor is the dominant pattern (7/9 apps).** Only Scrivener and Dabble default to first-line indent in the editor, and Scrivener explicitly encourages customizing this.

4. **Focus mode and typewriter scrolling are the features that signal "this is a serious writing tool."** They are the features writers cite when explaining why they chose a dedicated writing app over Google Docs or Word.

5. **The editor and the output are separate concerns.** Vellum, Atticus, Scrivener, Ulysses, and iA Writer all treat these independently. The editor should be optimized for writing comfort; the preview/export should be optimized for reading/publishing standards.

6. **16-18px font size with 1.5x line height is the sweet spot** for extended screen writing. Smaller sizes (Scrivener 13pt, Google Docs 11pt) are designed for document creation, not hours of creative writing.
